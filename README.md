# DocFlow AI — 100% Local Intelligent Document Processing Platform

Every component runs on your machine. No API keys, no cloud calls, no data leaving your device.

| Capability            | Local technology used                                  |
|------------------------|---------------------------------------------------------|
| OCR / text extraction  | Tesseract (`pytesseract`) + poppler (`pdf2image`)       |
| Generic doc parsing    | Apache Tika (runs as a local JAR)                       |
| LLM (Q&A, classification, extraction) | Ollama running `llama3.1` (or any local model) |
| Embeddings             | `sentence-transformers` (`all-MiniLM-L6-v2`, on-device) |
| Vector search           | ChromaDB (persisted to local disk, embedded mode)       |
| Structured storage      | SQLite via SQLAlchemy (swap for local Postgres if you like) |
| Web framework           | FastAPI + a static HTML/JS frontend                      |

## 1. System prerequisites (install once)

```bash
# macOS
brew install tesseract poppler openjdk

# Ubuntu/Debian
sudo apt-get install -y tesseract-ocr poppler-utils default-jre
```

Install and start [Ollama](https://ollama.com), then pull a model:

```bash
ollama pull llama3.1
ollama serve   # keep this running in the background
```

## 2. Python setup

```bash
cd docflow-ai
python3 -m venv venv
source venv/bin/activate         # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env             # adjust paths/model names if needed
```

## 3. Run

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

Open **http://localhost:8000** for the built-in UI, or **http://localhost:8000/docs** for the interactive API (Swagger UI).

## 4. Typical flow

1. `POST /documents/upload` — upload any PDF/image/docx/xlsx. Text is extracted locally immediately.
2. `POST /documents/{id}/process` — runs the local workflow: classify → extract structured fields → index into ChromaDB.
3. `POST /qa` — ask natural-language questions; answers are generated by your local Ollama model using retrieved chunks (RAG).
4. `POST /search` — pure semantic search across all indexed documents.

## 5. Swapping SQLite for local Postgres

Run Postgres locally (e.g. via Docker), then update `.env`:

```
DATABASE_URL=postgresql://user:password@localhost:5432/docflow
```

and add `psycopg2-binary` to `requirements.txt`.

## 6. Notes on "100% local"

- Tika's Python wrapper auto-downloads a Tika server **jar** on first run and then talks to it over `127.0.0.1` — no further network access needed afterward.
- `sentence-transformers` downloads model weights once from Hugging Face on first use; after that it runs fully offline.
- Ollama downloads model weights once (`ollama pull`), then all inference is local.
- If you need to run in a fully air-gapped environment, pre-download the Tika jar, the embedding model, and the Ollama model on a connected machine and copy them over.
